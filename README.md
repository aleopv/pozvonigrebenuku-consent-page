# Страница согласий «Позвони Гребенюку»

Самостоятельная страница документов и согласий перед оплатой подписки.

Она не связана с подтверждением номера и может открываться из Telegram-бота сразу после экрана стоимости подписки.

## Пользовательский flow

```text
Telegram: стоимость подписки
→ страница документов и согласий
```

Переходы:

- «Продолжить» — статичная кнопка без сохранения, отправки данных и перехода в бот;
- логотип → стартовый экран `@grebenyuk_call_bot`;
- ссылки на документы встроены в соответствующие строки согласий.

Чекбоксы:

- обязательный единый чекбокс: Оферта + Политика конфиденциальности;
- обязательный отдельный чекбокс: согласие на обработку персональных данных;
- обязательный отдельный чекбокс: согласие на запись и обработку голосового разговора;
- обязательный отдельный чекбокс: согласие на обработку разговора с помощью ИИ;
- обязательный отдельный чекбокс: согласие на автоматическое периодическое списание;
- необязательный отдельный чекбокс: согласие на рекламные и информационные сообщения.

Кнопка «Продолжить» активируется только после всех обязательных чекбоксов. Рекламный чекбокс не предустановлен и не влияет на активность кнопки.

## Настройка

Параметры находятся в `config.js`:

- `price` — стоимость подписки;
- `defaultBackUrl` — Telegram deep link логотипа;
- `documents` — URL и версии документов.

### Документы

PDF-файлы документов кладутся в папку `static/`. Версия документа обязательно указывается прямо в имени файла, например:

```text
static/offer-v2026-06-29.pdf
static/privacy-v2026-06-29.pdf
static/subscription-v1.0.pdf
```

После добавления PDF обновите соответствующую запись в `config.js`:

```js
offer: {
  url: "./static/offer-v2026-06-29.pdf",
  version: "v2026-06-29"
}
```

Сейчас production PDF подключены из `static/`. Для `recording` и `aiConversation` используется один документ о записи и обработке голосового разговора.

## Текущий режим

Сейчас страница не отправляет данные на backend. Кнопка «Продолжить» нужна только как визуальная проверка: она остаётся неактивной, пока пользователь не отметит все обязательные согласия.
