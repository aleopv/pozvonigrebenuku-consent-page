# Страница согласий «Позвони Гребенюку»

Самостоятельная страница документов и согласий перед оплатой подписки.

Она не связана с подтверждением номера и может открываться из Telegram-бота сразу после экрана стоимости подписки.

## Пользовательский flow

```text
Telegram: стоимость подписки
→ страница документов и согласий
→ Telegram: согласия подтверждены
→ переход к оплате
```

Переходы:

- «Продолжить» → `@grebenyuk_call_bot`, состояние `payment_ready`;
- «Назад» → стартовый экран `@grebenyuk_call_bot`;
- «Документы» → модальное окно со списком юридических документов.

## Настройка

Параметры находятся в `config.js`:

- `price` — стоимость подписки;
- `submitUrl` — endpoint сохранения согласий;
- `defaultReturnUrl` — Telegram deep link после успешного сохранения;
- `defaultBackUrl` — Telegram deep link кнопки «Назад»;
- `documents` — URL и версии документов.

Сейчас для теста все документы используют одну временную Google Drive-ссылку. Перед production-запуском её нужно заменить отдельным актуальным URL для каждого документа.

Страница принимает необязательные query-параметры:

- `session_id` — идентификатор сессии оформления;
- `user_id` — идентификатор пользователя;
- `telegram_webapp=1` — передать результат через Telegram WebApp `sendData`.

## Backend-контракт

При заполненном `submitUrl` отправляется `POST application/json`:

```json
{
  "type": "consent_accepted",
  "nextStep": "payment_ready",
  "sessionId": "abc123",
  "userId": "42",
  "acceptedAt": "2026-06-19T12:00:00.000Z",
  "documents": {
    "offer": "2026-06-19"
  },
  "consents": {
    "offer": true,
    "personalData": true,
    "recording": true,
    "aiProcessing": true,
    "recurringPayment": true,
    "marketing": false
  }
}
```

В production backend должен:

1. проверить Telegram-пользователя и сессию;
2. сохранить версии документов, обязательные согласия и время принятия;
3. вернуть `returnUrl` либо подготовить состояние `payment_ready` в основном боте;
4. не разрешать оплату без серверной записи обязательных согласий.

Пока `submitUrl` пустой, страница работает как демонстрация: сохраняет payload в `localStorage` и переходит в основной бот.
